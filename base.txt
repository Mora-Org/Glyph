Documento de Especificação: Gerador de Ensaios Visuais (Desktop)
1. Visão Geral do Sistema
Um software desktop open-source focado na criação de ensaios visuais e apresentações de alta customização estética. O diferencial é o controle granular sobre tipografia, cenas e hardware, dispensando a complexidade de editores de vídeo tradicionais (Premiere/DaVinci) em favor de um fluxo de trabalho focado em assets (imagens, gifs, vídeos) e cenas programáveis.

2. Stack Tecnológico Sugerido (Desktop-First)
Como você mencionou o Obsidian, precisamos de tecnologias de interface web rodando nativamente no desktop.

Desktop Wrapper: Tauri (altamente recomendado). É construído em Rust, absurdamente mais leve e rápido que o Electron, e permite usar o frontend web que você já domina.

Frontend (Interface UI): React ou Next.js (em modo Static Export).

Manipulação de UI/Drag & Drop: @dnd-kit/core (para mover elementos na tela livremente) e fabric.js ou Konva.js (excelentes bibliotecas de Canvas 2D para desenhar, rotacionar e arrastar as imagens e textos).

Motor de Renderização (Backend Local): FFmpeg. É o padrão ouro. O Tauri pode executar comandos do FFmpeg nativamente no seu sistema operacional para processar os vídeos com precisão de frames.

3. Core Features & Estrutura
A. Gerenciamento de Assets e Projetos
Dashboard: Tela inicial com os projetos recentes.

Explorador Local: Leitura direta de pastas do sistema do usuário, sem necessidade de upload.

Organização: Suporte a separação nativa de Fontes (.ttf, .otf), Imagens, Vídeos e GIFs.

B. Motor de Cenas (Timeline e Canvas)
Estrutura Sequencial: O projeto é uma lista de objetos Cena. Cada Cena possui sua própria duração (duração_total: X segundos).

Controle de Elementos (Canvas): * Interface visual (drag & drop) para posicionar, redimensionar e sobrepor elementos.

Cada elemento (imagem, texto) dentro da cena possui uma linha do tempo própria (ex: a cena dura 10s, mas a Imagem A aparece do segundo 2 ao 8).

Transições e Pausas:

Definição de corte entre a Cena A e Cena B (Corte seco, Fade, etc.).

Gerador de "Pausas Ativas": Inserção de blocos de tempo vazios configuráveis (Ex: 5 segundos de tela preta, ruído estático/VHS, ou cor sólida).

C. Manipulação Avançada de Tipografia
Controle por Caractere (Lettering Dinâmico): Capacidade de quebrar uma string em caracteres individuais.

Permitir que o usuário mova o eixo X/Y de uma única letra (ex: sobrepor o "O" no "D").

Permitir tamanhos, cores e fontes diferentes dentro da mesma palavra.

Efeitos Visuais em Textos:

Clipping Mask (Vídeo no Texto): Usar o texto como máscara de recorte para que um vídeo seja reproduzido apenas "dentro" das letras.

Animações Nativas: Efeitos pré-programados no Canvas (Tremor de ansiedade, piscar como placa de neon de Las Vegas, estilo letreiro de quadrinhos).

D. Sistema de Filtros e Efeitos Orgânicos
Aplicação de máscaras em imagens (recorte estilo jornal rasgado, bordas irregulares, remoção de fundo).

Sugestão de Lib: Filtros CSS nativos no frontend para o preview, traduzidos para filtros complexos do FFmpeg na hora da renderização.

E. Reusabilidade (Common Places)
Sistema de templates. Uma cena altamente customizada pode ser salva em uma biblioteca local de Componentes/Cenas.

O usuário pode instanciar essa cena em qualquer projeto futuro, mantendo a consistência estética do canal.

F. Motor de Exportação e Hardware
Seletor Automático de Perfil: O software escaneia o hardware local e sugere a melhor pipeline de renderização.

Controle Manual do Usuário:

Single Thread (mais lento, não trava o PC para outras tarefas).

Multithread (usa toda a CPU).

Aceleração via GPU (NVENC para Nvidia, AMF para AMD).

Cross-platform: FFmpeg garante compatibilidade perfeita entre Windows e Linux na hora de exportar o .mp4 ou .mov.
